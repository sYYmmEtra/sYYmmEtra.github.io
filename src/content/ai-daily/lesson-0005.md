# 📅 2026-07-09 · 轨道B · 梯度下降、反向传播与优化器（SGD / Adam / LR 调度）· 深度 L2

> **TL;DR**：训练神经网络＝**反向传播算梯度**（就是链式法则在计算图上跑一遍）＋**优化器拿梯度更新权重**。SGD 是"沿梯度反方向走一小步"；动量给它加惯性；Adam 在动量之外再给**每个参数配一个自适应步长**（用梯度平方的滑动平均做分母）。这三样构成了今天几乎所有大模型训练的默认底座，而学习率调度决定了"步子多大、什么时候收"。

---

## 一、核心概念精讲（L2）

我们把"训练"拆成三个环环相扣的问题，逐个讲透：**(1) 用什么目标？→ 损失与梯度；(2) 梯度怎么算出来？→ 反向传播；(3) 拿到梯度怎么更新？→ 优化器 + 学习率调度。**

### 1. 起点：为什么是"沿梯度下降"

训练的本质是解一个优化问题：找一组参数 θ，让损失函数 L(θ) 最小。损失（loss）就是"模型预测有多错"的标量度量，比如语言模型用交叉熵。

梯度（gradient，∇L）是 L 对每个参数的偏导数拼成的向量。它的关键性质：**∇L 指向 L 上升最快的方向**。所以想让 L 下降，就往反方向走：

```
θ ← θ − η · ∇L(θ)
```

这就是**梯度下降（Gradient Descent）**。η（eta）是**学习率（learning rate）**，控制步长。

> **直觉类比**：你蒙眼站在山坡上想下到谷底。你没有全局地图，但能用脚感受**当前脚下哪个方向最陡**（这就是梯度）。于是你朝最陡的下坡方向迈一步，重新感受，再迈一步。η 就是你每步迈多大——迈太小走一整天，迈太大可能一脚跨过谷底摔到对面坡上。

**为什么不直接解析求解（令 ∇L=0）？** 因为神经网络的 L 是千亿参数的高度非凸函数，没有闭式解；而且数据太多，一次性算全量梯度太贵。所以我们用**迭代 + 采样**来逼近。

### 2. SGD：为什么"随机"反而是特性

严格的梯度下降要用**全部训练数据**算一次 ∇L，才走一步。数据集上亿条时这不可行。

**随机梯度下降（SGD, Stochastic Gradient Descent）**的做法：每次只从数据里抽一个 **mini-batch**（比如 256 条），用这批数据的平均梯度当作全量梯度的**无偏估计**，走一步。

```
每步：
  抽一个 batch B
  g = (1/|B|) Σ_{x∈B} ∇L(θ; x)   # 这批的平均梯度
  θ ← θ − η · g
```

- **为什么这样也能收敛？** 因为 batch 梯度的期望等于真实梯度（无偏），大量小步的噪声会相互抵消，整体朝正确方向走。
- **"随机"是 bug 还是 feature？** 是 feature。噪声让优化器能**跳出尖锐的坏极小值**，倾向于停在平坦区域——而平坦极小值通常泛化更好。这是 SGD 至今在某些视觉任务上仍打得过花哨优化器的原因之一。
- **常见误区**：以为 batch 越大越好。batch 越大梯度越准、越省通信，但噪声减少会损害泛化，且存在"**临界 batch size**"——超过它继续加大，收敛所需步数几乎不再下降，纯浪费算力。（这个概念在今天的最新动态里会再出现。）

### 3. 反向传播：梯度到底怎么算出来的

上面所有公式都假设"我们有 ∇L"。但一个几十层的网络，怎么高效算出损失对**每一个**参数的偏导？答案是**反向传播（Backpropagation）**——它不是一个新算法，就是**链式法则（chain rule）在计算图上的系统化应用**。

把网络看成一张**计算图**：每个节点是一个运算（矩阵乘、加偏置、激活函数……），数据从输入正向流到损失（**前向传播 forward pass**），每个节点顺手记住自己的输入。

反向传播分两步走：

1. **前向**：正常算一遍，得到最终 loss，并缓存每层的中间值（激活值）。
2. **反向**：从 loss 出发，**从后往前**逐节点算梯度。核心是链式法则——如果 `L` 依赖 `y`，`y` 依赖 `x`，那么 `∂L/∂x = (∂L/∂y)·(∂y/∂x)`。每个节点收到"上游传来的梯度 ∂L/∂y"，乘上"自己这个运算的局部导数 ∂y/∂x"，就得到要继续往下游传的梯度。

> **直觉类比**：一条流水线出了次品（loss 大），要追责。你从终点往回问："这个瑕疵，最后一道工序贡献了多少？"——它把责任按比例（局部导数）分给它的每个上游输入，上游再继续往它们的上游分。**每个环节只需要知道"上游甩给我多少责任"和"我自己这步的敏感度"**，就能算出自己的那份，并正确地把剩下的责任继续往回传。

**为什么反向传播是"神来之笔"？** 关键在**复杂度**。如果对每个参数单独用"扰动一下看 loss 变多少"（数值微分）来估梯度，N 个参数就要跑 N 次前向，千亿参数直接爆炸。而反向传播利用了**中间结果可复用**：一次前向 + 一次反向，就能同时得到**所有**参数的精确梯度，总代价约等于前向的 2 倍，与参数量无关地"一趟搞定"。这就是为什么深度学习能规模化。

- **工程代价——显存**：反向要用到前向缓存的所有激活值，所以**激活值占的显存**常常比参数本身还大。这催生了 **gradient checkpointing**（丢弃部分激活、反向时重算，用计算换显存）等技术。
- **常见误区**：把反向传播和"梯度下降"混为一谈。反向传播只负责**算梯度**，梯度下降/SGD/Adam 才负责**用梯度更新**。前者是"求导引擎"，后者是"更新策略"，PyTorch 里正好对应 `loss.backward()` 和 `optimizer.step()` 两行。

### 4. 动量：给 SGD 加惯性

纯 SGD 的毛病：在"细长峡谷"型的损失面上，它会在陡峭的两壁间来回横跳（震荡），沿谷底方向却挪得很慢。

**动量（Momentum）**的修法：不直接用当前梯度更新，而是维护一个梯度的**指数滑动平均（EMA）**速度向量 v：

```
v ← β·v + (1−β)·g        # β 常取 0.9，v 是"惯性速度"
θ ← θ − η·v
```

- **为什么有效**：来回震荡的方向，正负梯度在 EMA 里相互抵消；而谷底那个持续一致的方向会被**累加放大**。等价于给小球加了惯性和阻尼，直冲谷底而不撞墙。
- 这是从 SGD 通往 Adam 的关键一步——Adam 的"一阶矩"就是这个动量。

### 5. Adam：为每个参数配一个自适应步长

动量解决了"方向"，但所有参数还是共享同一个 η。问题是：不同参数的梯度尺度差异巨大（有的梯度经常很大，有的很小），一个统一步长必然对某些参数太大、对另一些太小。

**Adam（Adaptive Moment Estimation）** 同时维护两个滑动平均：

- **一阶矩 m**：梯度的 EMA（就是动量，管方向）
- **二阶矩 v**：梯度**平方**的 EMA（管每个参数各自的"典型幅度"）

```
m ← β1·m + (1−β1)·g          # 一阶矩，β1≈0.9
v ← β2·v + (1−β2)·g²         # 二阶矩，β2≈0.999，g²是逐元素平方
m̂ = m / (1−β1^t)             # 偏差校正（bias correction）
v̂ = v / (1−β2^t)
θ ← θ − η · m̂ / (√v̂ + ε)     # 逐元素相除！每个参数有效步长不同
```

拆开看这个更新式的精髓——**`m̂ / √v̂`**：

- 分子 `m̂` 是平滑后的方向（动量）。
- 分母 `√v̂` 是"这个参数近期梯度有多大"。梯度一直很大的参数，分母大 → 步子被**自动缩小**；梯度一直很小的参数，分母小 → 步子被**放大**。
- 结果：**每个参数都得到了自己的自适应学习率**，尺度被自动归一化。这让 Adam 对学习率不那么敏感、开箱即用，是它统治大模型训练的根本原因。

**几个必须讲清的细节：**

- **偏差校正（bias correction）**：m、v 初始化为 0，训练最初几步 EMA 严重偏向 0（低估）。除以 `(1−β^t)` 把这个系统性低估补回来（t 大时该因子趋近 1，校正自动消失）。少了它，训练初期步长会异常小。
- **ε（epsilon）**：分母加的一个小常数（如 1e-8），纯粹防止除零和数值爆炸，不是超参重点。
- **AdamW vs Adam——今天大模型实际都用 AdamW**：原始 Adam 把 L2 正则（权重衰减）混进梯度里，会被二阶矩 `√v̂` 缩放，导致正则强度对不同参数不一致。**AdamW（Decoupled Weight Decay）**把权重衰减从梯度里**解耦**出来，单独作用：`θ ← θ − η·(m̂/√v̂ + λ·θ)`。这个看似微小的改动显著改善了泛化，是现在的事实标准。**常见误区**：以为 Adam=AdamW，其实你训 LLM 时用的几乎都是后者。
- **代价**：Adam 要为每个参数存 m 和 v 两份状态，**优化器状态显存 = 2× 参数量**（fp32 下更多）。这是大模型训练显存的大头之一，也是后面 Muon 主打"省显存"的对比基准。

### 6. 学习率调度：步子不能一成不变

固定 η 是次优的：训练初期想快、后期想稳。**学习率调度（LR schedule）**动态调整 η，现代 LLM 训练的标配组合是：

- **Warmup（预热）**：前几百~几千步把 η 从 0 **线性升到**峰值。**为什么需要**：训练初期参数是随机的，梯度方向噪声大、二阶矩估计还不准，一上来就大步走极易发散（尤其 Transformer + Adam）。先小步"热身"让统计量稳定。
- **Decay（衰减）**：之后让 η 缓慢下降，常用**余弦退火（cosine decay）**——按余弦曲线平滑降到接近 0。**为什么**：后期在极小值附近，大步会反复越过最优点来回震荡；小步才能精细收敛、"坐稳"到谷底。

典型曲线（ASCII 示意）：

```
η
│      ╭──────╮
│     ╱        ╲___
│    ╱             ╲____
│   ╱                   ╲___
│  ╱(warmup)   (cosine decay)╲__
└─┴────────────────────────────── step
```

> **一句话记住**：warmup 防"冷启动炸膛"，cosine decay 防"终点前刹不住车"。

### 🔎 完整走一遍：一步训练里发生了什么

拿一个极简例子把上面全部串起来。设一个只有一个权重 `w` 的线性模型 `ŷ = w·x`，损失 `L = (ŷ − y)²`，用一条数据 `x=2, y=6`，当前 `w=1`，学习率 `η=0.1`，优化器先看 SGD 再看 Adam。

**Step 1｜前向传播**
- `ŷ = w·x = 1·2 = 2`
- `L = (ŷ − y)² = (2 − 6)² = 16`

**Step 2｜反向传播（链式法则）**
- `∂L/∂ŷ = 2(ŷ − y) = 2·(2−6) = −8`（上游梯度）
- `∂ŷ/∂w = x = 2`（本地导数）
- 链式相乘：`∂L/∂w = (∂L/∂ŷ)·(∂ŷ/∂w) = (−8)·(2) = −16`
  → 梯度 `g = −16`（负号说明：增大 w 会让 L 下降，所以应该增大 w）

**Step 3｜优化器更新**

用 **SGD**：
- `w ← w − η·g = 1 − 0.1·(−16) = 1 + 1.6 = 2.6`
- w 从 1 涨到 2.6，朝着让 `ŷ` 逼近 6 的方向走对了。✅

用 **Adam**（首步 t=1，β1=0.9, β2=0.999, ε≈0）：
- 一阶矩 `m = (1−0.9)·(−16) = −1.6`；二阶矩 `v = (1−0.999)·(−16)² = 0.256`
- 偏差校正 `m̂ = −1.6/(1−0.9) = −16`；`v̂ = 0.256/(1−0.999) = 256`
- 更新 `w ← 1 − 0.1·(−16/√256) = 1 − 0.1·(−16/16) = 1 − 0.1·(−1) = 1.1`

**关键洞察**：注意 Adam 首步的 `m̂/√v̂ = −16/16 = −1`——它把梯度**归一化到了 ±1 量级**！所以 Adam 首步几乎总是走"η 这么大的一步"，与梯度绝对大小（这里是 16）无关。这正是 Adam 对梯度尺度不敏感、鲁棒好调的机制来源。SGD 一步跨了 1.6，Adam 只跨了 0.1，更保守稳健——代价是这一步"信息量"用得更省。

> ⚠️ 本次 WebFetch 被网络策略拦截，无法逐篇打开 arXiv 原文核对摘要；以下条目均在**两次独立 WebSearch 结果中交叉出现**（编号、标题、机构一致），可信度较高，但请你点开链接二次确认后再深读。**主线：2024 年底至今，"Adam 一统天下"的格局正被 Muon 撬动。**

1. **Practical Efficiency of Muon for Pretraining** · arXiv:2505.02222（Essential AI）· 2025
   - 为什么重要：把 **Muon** 明确定位为"最简单的二阶优化器"，并实证它在 compute-time 权衡上**全面超越 AdamW**——尤其在**大 batch（远超临界 batch size）**下更能保住数据效率。还解决了 muP 超参迁移问题，首次演示用 muP 为 Muon 训练的 LLM 校准超参。
   - 与今日主题的关系：直接接续我们讲的"SGD/Adam + 临界 batch size"。Muon 不再把每个参数当独立标量（Adam 的做法），而是认识到**权重是有几何结构的矩阵**，对梯度矩阵做**正交化**（Newton-Schulz 迭代）后再更新，省显存约 33%、收敛更快。

2. **Muon is Scalable for LLM Training** · arXiv:2502.16982（Moonshot AI / Kimi 团队）· 2025
   - 为什么重要：回答了"Muon 能否放大到真实大规模训练"这一关键问题——答案是能。Muon 已被用于训练 **Kimi K2、GLM-5** 等一线开源模型，业界正把它当作新的默认优化器之一（如 nanochat）。
   - 与今日主题的关系：这是"优化器"这条主线从教科书（SGD/Adam）走到工业前沿的活证据；说明优化器的选择直接影响千万美元级的训练成本。

3. **The Potential of Second-Order Optimization for LLMs: A Study with Full Gauss-Newton** · arXiv:2510.09378 · 2025
   - 为什么重要：给出"上限参照"——用完整 Gauss-Newton 二阶法，达到同等 loss 只需约 Muon 的 **1/16 步数**；但每步计算太贵，暂不实用。这划定了"优化器还能进步多少"的天花板。
   - 与今日主题的关系：帮你把 SGD（纯一阶）→ Adam（对角近似二阶）→ Muon/Shampoo（更强的矩阵结构近似）→ 完整二阶 这条**"越来越会用曲率信息"的谱系**串成一条线。

> 一句话趋势判断（我的推断，非共识）：Adam/AdamW 短期仍是安全默认，但**矩阵感知优化器（Muon 为代表）正在成为大模型预训练的新前沿**，值得作为轨道 C 的专题深挖。
